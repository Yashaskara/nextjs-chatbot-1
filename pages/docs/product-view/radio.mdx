# Product View Buttons

## Overview

The SDK includes a feature that allows users to customize the product display on the Search Results Page (SRP) by selecting from various view options. For example, if the user provides options such as "grid" and "list," the products will be arranged accordinglyâ€”displayed in a grid layout when "grid" is selected, or in a list format when "list" is chosen. This flexibility ensures that the product presentation aligns with the user's preferences.

## How it Works ?

When the user selects an option from the product view, the state is updated with the corresponding view. The Products component then applies this view from the state as a class to the wrapper, allowing CSS to adjust the appearance and alignment of the products accordingly.

## Usage

```js copy
// importing component
import { ProductViewRadioButtons } from "@unbxd-ui/react-search-components";

// import the style for the component
import "@unbxd-ui/react-search-components/styles/productViewRadioButtons.css";
```

If the styles import does not work, use the following code to import it:

```js copy
require.resolve("@unbxd-ui/react-search-components/styles/productViewRadioButtons.css");
```

## Props

<div style={{ position: "relative" }}>
	### CustomComponent
	<Pills pills={[{ type: "type", label: "component" }]} left={"240px"} />
</div>

This component renders each button and receives the clicked option in its props as `item`. For e.g., `{label: "GRID", value: "GRID"}`. <br></br>
Example component:

```js
const CustomComponent = (props) => {
	const { item } = props;
	return item.label;
};
```

<div style={{ position: "relative" }}>
	### showLabel
	<Pills
		pills={[
			{ type: "required", label: "required" },
			{ type: "type", label: "boolean" },
		]}
		left={"145px"}
	/>
</div>

A boolean value which indicates if the label must be shown or not. By default, the value is `true`.

<div style={{ position: "relative" }}>
	### label
	<Pills pills={[{ type: "type", label: "string" }]} left={"80px"} />
</div>

The text that must be shown as the label to the view component.

<div style={{ position: "relative" }}>
	### options
	<Pills
		pills={[
			{ type: "required", label: "required" },
			{ type: "type", label: "array" },
		]}
		left={"110px"}
	/>
</div>

The options that must be displayed in the dropdown.
The options must be of the form:

```js copy
options={[
	{
		value: "GRID",
		label: "GRID",
		classname: "gridClass"
	},
	{
		value: "LIST",
		label: "LIST",
		classname: "listClass"
	}
]}
```

<Callout>
	<div>Note:</div>
	<div>The classname provided in the options is applied as a class to the products wrapper.</div>
</Callout>

	### defaultView
	<Pills pills={[{ type: "type", label: "string" }]} left={"150px"} />
</div>

The initial value of view that must be applied to the products.

<div style={{ position: "relative" }}>
	### styles
	<Pills
		pills={[
			{ type: "optional", label: "optional" },
			{ type: "type", label: "object" },
		]}
		left={"95px"}
	/>

This prop will be used to style the component. The value must use a defined structure (the structure can be seen in the following example).
The keys in the structure must be the same while the values can changed as per the user's choice.

```css copy
styles={{
    wrapper: "view-root",
    label: "view-label",
    container: "view-container",
    radioItem: "view-radio-item",
    inputBtn: "view-button",
    inputLabel: "view-input-label",
    selected: "selected",
}}
```
